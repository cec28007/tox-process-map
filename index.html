Product Code (a.k.a. Test Code)Pre-defined catalog (TX0xxx)Each line item on the quoteIdentifies what test (GPMT, MEM elution, pyrogen, etc.)
GLP NumberTest Facility ManagementWhen study is assigned (GLP Assignment List)Identifies the study as a GLP study; may not be needed long-term

Service Buckets

Three standardized buckets + one placeholder for custom work.

Bucket A — BET (Bacterial Endotoxins)

  • One line item = one test = one result.
  • ROC pools; SLC runs individually.
  • Destructive; extraction per PYT-8 or method validation (see QA-396 §B).
  • Simple case for LIMS.

Bucket B — Cytotox (MEM Elution / Direct Contact)

  • Structurally same as BET for intake/billing.
  • Used as screening AND as a line inside biocomp packages.
  • Extraction per QA-396 §B (Elution) or §C (Direct Contact).

Bucket C — Biocompatibility (the complex one)

Typical "Big 5":

  1. Intracutaneous Irritation
  2. Systemic Injection
  3. Guinea Pig Maximization Test (GPMT)
  4. MEM Elution / Cytotox
  5. Material-Mediated Pyrogen

Resource sharing: ~13 devices typical → 6 for top 3 (share polar + non-polar × 3 time points) + 1 for MEM + 5 for pyrogen + 1 spare.

Each test still: has its own product code, protocol, invoice line, and report.

Bucket D — Complex / Custom Studies [PLACEHOLDER]

  • 8–10/yr (P&G, PopBio, Idera, etc.).
  • Heavy change orders; milestone-based billing already in use.
  • Out of scope for v1 detail — shown on the map as a lane; process to be captured later.

LIMS Modeling Implications

First-cut observations to feed the LIMS data model.

  1. Shared test-article pool per study — LIMS needs a "test article inventory" concept where one receipt feeds multiple product-code tests. Lindsey World calls this S-number → child-sample tree; paper analog today is TAD sheet QA-2.
  2. Time-point-based sub-sampling — extracts are recreated at t=0/14/21 (not aliquoted from a frozen stock). LIMS must model "extract creation event" as a distinct record per time point.
  3. Animal traceability bridge — in-vivo endpoints must link to Animal Tracks records. [?] Animal Tracks is homegrown; API should be feasible.
  4. Environmental monitoring bridge — Rees probe data should be pullable into the study record. [?] Rees API/export format TBD.
  5. IACUC protocol link — each study carries an IACUC protocol # (separate from the study protocol). [?] confirm whether captured as a field today.
  6. Endpoints vary by protocol (confirmed) — endpoint schema must be protocol-driven, not product-code-driven. Templated + study-specific forms both exist today.
  7. Complex studies — keep as a separate study type with a minimal data model; don't force into the standardized flow in v1.
